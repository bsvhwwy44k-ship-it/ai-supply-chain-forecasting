# Architecture

## Design Philosophy

This framework treats weekly business metrics reporting as a pipeline problem:

```
Raw Data → Normalization → Computation → Report Generation
```

Each stage is a standalone script that reads from files and writes to files. The AI layer (Claude Code) orchestrates the pipeline conversationally — you describe what you want, it runs the right sequence.

## Components

### Data Layer
- **S3 Sync** — Pulls latest exports from cloud storage with date filtering
- **CSV Ingest** — Normalizes downloaded files into consistent naming/location
- **Query Prep** — Parameterizes SQL templates with week/date/region, copies to clipboard

### Analysis Layer
- **Bridging Engine** — Computes WoW and YoY deltas for ratio metrics (numerator/denominator)
- **Driver Analysis** — Linear regression decomposition of a composite metric into its drivers
- **Forecast Comparison** — Merges multiple planning cycles with actuals, quantifies miss in bps

### Output Layer
- **Markdown Reports** — Tables formatted for Obsidian/GitHub rendering
- **PDF Generation** — Multi-page reports with tables and charts
- **Regional Maps** — Choropleth US maps with metric annotations

## Key Design Decisions

### Why separate SQL files?
Queries change frequently (new metrics, date ranges, marketplace filters). Keeping them in `.sql` files means:
- Non-engineers can review/edit queries
- Git diffs show exactly what changed
- Python runner stays stable while queries evolve

### Why ratio metrics (numerator/denominator)?
Most supply chain KPIs are ratios: units/box, units/purchase, %instock. Storing raw numerator and denominator allows:
- Proper aggregation (sum numerators, sum denominators, then divide)
- Weighted averages across regions/time periods
- Drill-down without losing precision

### Why ISO weeks?
The reporting calendar aligns with ISO weeks. Week 1 starts on the Monday of the first week containing Thursday of the year. This ensures:
- Consistent 7-day periods
- Proper YoY comparison (same week number)
- No partial-week edge cases

### Why Obsidian vault?
- WikiLinks create a knowledge graph naturally
- Markdown is version-controllable
- Reports generated by scripts are immediately viewable
- No vendor lock-in — it's just folders and `.md` files
